# Codebase Audit — clawdcode (consolidated)

## Executive summary

Overall health is **mixed-but-recoverable**: the core pipelines (hook queue, shared/ split, session policy) are clean and well-factored, but the codebase is dominated by two structural anti-patterns. First, a **massive duplication tax**: three chat-integration files (telegram/discord/slack, ~5,200 LOC) are the same program written three times, and three frozen web UI bundles (`web/app`, `web/os9`, `web/osish`, ~6,200 LOC, one commit each) are dead-on-the-vine parallel UIs nobody maintains. Second, a **cluster of ~1700-line god-modules** (`runner.ts`, `start.ts`, `server.ts`, each chat file) where mutable closure state and copy-pasted branches breed correctness bugs. The most urgent findings are real security/correctness gaps: a Slack prompt-injection hole (no `wrapUntrusted`), a CSRF bypass when `Origin` is absent, a shell-injection vector in `preflight.ts`, a Datadog `rule === true` denial-of-wallet, and several just-landed multi-repo migration bugs that silently drop jobs/routines. The fork has already produced measurable drift (capability/safety controls present on one platform, absent on others).

## P0 — correctness & security

1. **chat-integrations · `slack.ts:1065,1067,1076,1084` · Slack never wraps untrusted user input — prompt-injection hole.** Telegram/Discord wrap external strings in `wrapUntrusted(...)`; Slack doesn't import it at all. Raw user text, skill args, voice transcript, and doc names flow straight into the Claude prompt. `sanitizeUserInput()` only strips Slack's own directives and only on the bot path. Fix: import and wrap `cleanText`, `args`, `voiceTranscript`, doc names, and thread-history identically to the other two. **cheap**

2. **core-modules · `preflight.ts:172,302` · Shell-injection via repo URL.** `run(\`git clone --quiet "${repoUrl}" "${tempDir}"\`)` is `execSync` with no quoting safety; a `"`/`$()`/backtick in any URL executes arbitrary commands. Fix: argv form — `Bun.spawn(["git","clone","--quiet",repoUrl,tempDir])` / `execFileSync`. Same for `git rev-parse HEAD` calls. **cheap**

3. **web-server · `server.ts:197-205` · CSRF guard only fires when `Origin` is present.** `if (origin) { … }` — a state-changing request with no `Origin` header bypasses the check. `SameSite=Lax` cookie mitigates top-level cross-site POSTs, but the explicit layer is load-bearing for bearer/`?token=` and non-browser clients. Fix: for cookie-authed state-changing requests require same-origin `Origin` **or** `Sec-Fetch-Site: same-origin`; treat absent-Origin-with-cookie as suspect. **cheap**

4. **hooks · `datadog.ts:124` · `rule === true` matches every Datadog alert (denial-of-wallet/noise).** Sentry guards this exact case with a prod-only default; Datadog has no safe default, so a programmatic `true` fires on every monitor/priority/type/tag and can spawn agent runs. Fix: add `defaultDatadogRule()` mirroring `defaultSentryRule` (require at least a priority floor) and resolve `true` through it in both parser and receiver. **cheap**

5. **core-modules · `jobs.ts:170,176` · Multi-repo routines silently dropped.** `loadJobs()` reads only `getJobsDir()` (first dir), but the multi-repo feature exposes `getJobsDirs()` = `[...repoDirs, DEFAULT_JOBS_DIR]`. With 2+ `jobsRepos`, every repo after the first never schedules, and the local default dir is skipped whenever any repo is configured. `migrateTriggers.ts:83` already iterates `getJobsDirs()`, proving intent. Fix: loop over `getJobsDirs()`, dedupe by job name (first dir wins). **cheap**

6. **daemon · `start.ts:1473,1530,1553,1589` (+ retry map `1461`) · Shared per-`job.name` state corrupted by concurrent hook threads.** The runner serializes by `threadId` only (`runner.ts:269`), but `currentActiveJobs`/`jobLastResult`/`jobRetryState` are keyed on `job.name`. Two PRs handled by one job run concurrently and collapse onto a single entry: the first's `.finally` deletes the active marker while the second is live (Schedule shows "not running"), and `jobLastResult` is last-writer-wins. Fix: key active/last-result state on the unique `threadId`, project to `job.name` for the Schedule view. **risky**

7. **daemon · `start.ts:340-359` · `runOutcome` scans only last 5 stdout lines → mislabels `[skip]` as "ok".** Agents that emit trailing tool-metadata/JSON after `[skip]`, or print it to stderr, push the marker out of the window — mis-classifying the queue/PR outcome that 06bd674 was added to surface. Fix: anchored `^\[skip\]` search over the full trimmed output, or have the runner return a structured `skipReason`. **cheap**

8. **hooks · `shared/hookPayload.ts:74` (dup `match.ts:247-257,274-284`) · Exclude-only glob lists invert filter semantics.** Empty list returns `false` (deny-all) while an all-exclusions list returns `true` (allow-all-but-x), so deleting the last positive pattern silently flips a deny-filter to allow. The hand-rolled tag-matcher loops handle the empty/all-exclude case differently and can drift. Fix: one documented matcher with `defaultInclude` derived from "any positive patterns present"; route Datadog/PR tag loops through it. **risky**

9. **run-engine · `runner.ts:1211` · Fresh-session arg filter strips *every* arg equal to the session id.** `args.filter((a) => a !== "--resume" && a !== existing?.sessionId)` removes any arg coinciding with the session id (prompt fragment, agent name), not just the token after `--resume`. The correct `stripResume()` helper (line 180) already exists and is used at line 1262 — this is an inconsistency. Fix: use `stripResume()` in all three spots. **cheap**

10. **run-engine · `runner.ts` + `sessions.ts:19,43-46` · Global session cache is process-wide mutable state with no write serialization.** Per-thread queues run `execClaude` concurrently; a thread run and a global stream run interleave `loadSession → mutate → Bun.write` on the shared `current`, losing `messageCount`/`turnCount` increments and clobbering `lastUsedAt`. Fix: route all global-session mutations through one serialized writer, or drop the cache and read-then-write atomically (file is tiny). **risky**

11. **web-server · `server.ts:739-771` · `/api/jobs/file/auto-name` ignores the repo dir.** Every sibling route resolves via `resolveJobsDir(repoSlug)`, but auto-name hardcodes the default dir for `readJobFile`, the collision check (`getJobsDir()`), and `renameJobFile`. For a file in a repo clone it throws ENOENT (400) or, worse, renames a same-named file in the local dir. Fix: thread `repo`/`dir` through exactly like the sibling routes. **cheap**

12. **web-ui-legacy · `HookConfigEditor.tsx:408-417` · `emit` spreads stale `skipSelf` over the freshly-computed local.** `{ ...value }` re-introduces the old `skipSelf` into `next`, and `onChange({ skipSelf, ...next })` lets `next.skipSelf` override the local for `updateRule`/`addRule`/`removeRule`. Works today only because the local is derived from the same `value.skipSelf` — a latent trap. Fix: stop spreading `value`; pass only `pr`/`comments`, let `emit` own `skipSelf`, drop it from `next` typing. **cheap**

13. **jobsRepo · `jobsRepo.ts:551-566` · `legacyEmptyStatus()` returns an invalid `RepoStatus`.** Missing non-optional `kind` and `jobs`; compiles only because `tsconfig` has no `strict`. UI consumers switching on `.kind`/`.jobs` get `undefined`. Fix: add `kind: "git", jobs: 0`; separately enable `strict`/`strictNullChecks`. **cheap**

14. **daemon · `start.ts:1661` + `hookQueue.ts:220-237,391` · Coalesced retries apply the oldest message's attempt cap to brand-new work.** `priorAttempts = Math.max(...attempts)`; `claimThread` doesn't bump attempts (only `defer` does). A fresh delivery coalesced with one that burned 4 attempts is dropped to `fail` after a single combined failure and inherits the batch's backoff `notBefore`. Fix: split the batch per-id on completion, or clamp `priorAttempts` to min for the cap check while keeping max for backoff timing. **risky**

15. **discord · `discord.ts:474-512,493,901` · `classifyThreadIntent` runs synchronous `execSync("claude … --print")` on every short guild message.** Blocks the single-WebSocket event loop up to 15s (can trip "Heartbeat not acked, reconnecting" at 1346), and is an unbounded cost amplifier — a full model call per short message before auth-gated routing. Not shell-injectable (input via `input`). Fix: async `Bun.spawn`/`execFile`, gate behind opt-in config, move off the hot path. **risky**

## P1 — structural / DRY

1. **chat-integrations (cross-cutting) · telegram/discord/slack (~5,200 LOC, 60-70% duplicated) · No shared platform abstraction.** `extractReactionDirective` is byte-identical in all three; `/status`/`/context` handlers, attachment download→transcribe→`promptParts` assembly, `sendMessage` chunking, typing-indicator, and the gateway/polling lifecycle are reimplemented per platform. The `shared/`+`messaging.ts` seam exists and is unused. Fix: a `ChatPlatform` interface (`send`/`sendTyping`/`react`/`downloadAttachment`/`triggerReason`/`sessionKey`) + a shared `handleInboundMessage(platform, msg)` orchestrator owning auth→dedup→wizard→rate-limit→attachment→`runUserMessage`→directive→reply. Platforms shrink to ~300-400-line adapters; retires ~2,500 lines and the security/capability drift below.

2. **web-bundles-dup (cross-cutting) · `web/app`, `web/os9`, `web/osish` (~6,243 LOC) · Three frozen parallel UIs.** Each has exactly one commit (May-26 daisyUI drop), zero edits since, while `web/ui` took 66 and `web/v3` took 10. CI builds them anyway; `web/api/*` changes silently rot them. They re-implement `web/ui` sections instead of wrapping them — the exact anti-pattern `web/v3` already solved. Fix: delete `web/app`/`web/index.tsx`/`web/os9`/`web/osish` + their `index.html`s, remove `darwin`/`os9`/`osish` from `build.ts` `bundles[]` and `BUNDLES` in `server.ts:267`. Keep at most one OS-themed pair behind `experimental/` with an owner. **(needs human decision)**

3. **web-server (cross-cutting) · `server.ts` (1665 lines) · Giant if-ladder `fetch` router.** ~40 endpoints matched inline by `pathname === …`/regex in one closure → no 405 handling (everything falls to bottom 404), ordering hazards (e.g. plugin route needs three `!==` guards), untestable. Fix: a route table (`{ method, pattern, handler }`) split into `src/ui/routes/{jobs,sessions,hooks,settings,plugins,chat,v3,mcp}.ts`; dispatcher does auth/host/CSRF once. Subsumes the body-parse / try-catch / SSE / dynamic-import duplication below; `server.ts` → ~150 lines of wiring.

4. **run-engine · `runner.ts` (1819 lines) · Central god-module.** Cleanly extractable: spawn/stream primitives → `claude-spawn.ts`; rate-limit singleton state → `rate-limit.ts`; env/path/security arg building → `spawn-config.ts`; agent-dir/CLAUDE.md → `agent-workspace.ts`. That alone drops it under 1k. Also: three near-identical NDJSON stream parsers (`runClaudeStream`/`runClaudeStreaming`/`streamClaude`) that have already drifted (timeout, plugin events) → one `parseClaudeStream(proc, {onText,onToolCall,…})` core; `execClaude`'s four sequential recovery branches sharing mutable locals → an ordered list of `(attempt) => attempt | null` strategies; session-id capture duplicated 6+ times → `persistNewSession()` + `sessionLabel()`.

5. **daemon · `start.ts` (1769 lines) · `start()` is a ~1140-line closure holding ~20 mutable vars + ~15 nested fns.** The closure-captured state is *why* P0-6 exists. Extract `parseStartArgs()` (pure), `STATUSLINE_SCRIPT` (114-line embedded JS → `.cjs` asset), and `src/daemon/{heartbeat,hookDrain,channels,jobRunner}.ts`. Also: `forwardToTelegram/Discord/Slack` are line-for-line triplicated (`1201-1253`) → one `forwardTo(channel,…)`; four "poll thread→session" helpers (`274-378`) → `withThreadSession()`; dual retry authorities with a `Number.MAX_SAFE_INTEGER` sentinel handshake (`1758`/`1595`/`1641`) → make `runJob` retry-agnostic; five inline `setInterval`s never cleared in `shutdown()` → collect handles for teardown.

6. **hooks · `match.ts` + four receivers · Match/skip-reason logic triplicated; receivers ~80% boilerplate.** Sentry/Datadog each have parallel `matchXRule`/`xRuleSkipReason` dimension-walks — the exact drift the PR-rule comment (`match.ts:104-108`) warns about; PRs were already fixed via `evalPrRule` returning `{ok, reason?}`. Fix: apply the `evalPrRule` pattern to Sentry/Datadog (also closes P0-4). The four receivers repeat content-type→`req.text()`→verify→parse→`Delivery`→`recordDelivery`→match-loop→evaluate; sentry/linear `verifyXSignature` are identical bar the header name. Fix: one `handleSignedWebhook(req, {verify, deriveId, deriveSummary, source, match})` envelope; model per-job decisions as a typed `RoutineOutcome`.

7. **chat-integrations · Feature/safety drift from the fork.** Telegram has `/mode`, `/kill`, rate-limit short-circuit (`isRateLimited()`), `isMainBusy()` guard; Discord and Slack lack most; Slack has no rate-limit check at all and will fan out subprocess runs while rate-limited. `/mode` (permission mode) and the busy/rate-limit guards are safety controls present on one platform only. Directive parsing is forked into 5+ divergent regex dialects (`[buttons:]` vs `[[slack_buttons:]]` etc.). Fix: the shared orchestrator (P1-1) makes these uniform by construction; one directive-extraction module returning normalized `{reaction, attachments, buttons, edits}`.

8. **web-server · Inline duplication the route table subsumes.** Body parse `(await req.json().catch(()=>({}))) as Record<string,unknown>` + per-field coercion ~20×; `withJson` (`http.ts:26`) used in exactly one place. `try{…}catch{ json({ok:false,error:String(err)},500) }` copy-pasted ~15×. `/api/chat` re-implements the SSE encoder/stream dance instead of `sseResponse()`. Per-request `await import(...)` 25+ times for stable internal modules. Fix: route everything through `withJson`, add a typed-body/schema helper, fold chat onto `sseResponse`, hoist stable imports.

9. **core-modules · Subprocess wrappers + frontmatter parsing duplicated.** Five near-identical subprocess wrappers (`runtime.ts:git()`, `jobsRepo.ts:runGit()`/`runClaude()`, `preflight.ts:run()`, `mcp.ts:spawnMcp()`) with inconsistent timeout/error handling → one `exec(argv, opts)` helper in `shared/`. Markdown-frontmatter/description extraction hand-rolled 3× (`skills.ts:84`, `slashRegistry.ts:22`, `jobsRepoPlugins.ts`) with different truncation limits → one `parseFrontmatter()`+`extractDescription()` reusing the `yaml` dep already in tree.

10. **jobsRepo · `jobsRepo.ts` · `slugForRepo` collision handling is inconsistent → two URL-colliding repos merge.** `ensureRepo`/`pullRepo`/`syncRepo`/`getRepoStatus` recompute the slug with an *empty* `existingSlugs`, while `findRepoBySlug` uses a collision-avoidance `seen` set → distinct slugs in one place, identical everywhere else, so `perRepoState`/clone-dirs/status collapse the two repos. Fix: compute slug once at config-parse time, store on `JobsRepoConfig`, every consumer reads that field.

11. **web-ui-legacy · `SettingsSection.tsx` (1166 lines) + render-phase `setState` repeated 5×.** Eight self-contained panels in one file → split into `sections/settings/{ReposPanel,ModelPanel,HeartbeatPanel,GitIdentityPanel,AppearancePanel,InstalledPluginsCard}.tsx`. `RunsSection.tsx` (1009) — move its `buildRows`/`buildFacets`/`applyFilters`/`SORTERS` engine (338-546) to `sections/runs/model.ts`. The `if (data && data !== seenState){ setSeenState; setX }` render-phase idiom is copy-pasted across `ReposPanel`/`HeartbeatPanel`/`ModelPanel`/`GitIdentityPanel`/`JobsSection` → one `useServerSeed<T>(data, onSeed)` hook. Also `ReposPanel` autosave (`346-374`) resurrects deleted/edited rows via a stale-closure race — pick one source of truth (don't `setUrls` inside save and re-seed on reload).

12. **web-ui-legacy · Provider/trigger parsing duplicated across the section layer.** The `/Triggered by GitHub … for scope \`…\`/` regex appears in `RunsSection:572`, `JobsSection:550`, and provider-from-event mapping is re-derived in 5 components (`RunsSection.providerOf`, `humanizeHookTrigger`, `ChatSection.HookSummary`, `DeliveriesSection.sourceFromEvent`). Fix: hoist `providerOf(event)` + the legacy-firstMessage parser into one shared `refs.ts`. Same family on the web-bundles side: `QueueStatusBadge` "ported" (not imported) into `web/v3/components/SectionTree.tsx:12`, `fmtRelative`/`MessageBubble` redefined across bundles — promote into `shared/`.

## Recommended fix batches

**Batch A — Security/correctness quick wins (mostly cheap, ship first).** P0-1 Slack `wrapUntrusted`, P0-2 preflight shell-injection, P0-3 CSRF Origin gap, P0-4 Datadog `rule===true`, P0-9 `stripResume` consistency, P0-11 auto-name repo dir, P0-13 `legacyEmptyStatus`. Spread across files but each is small and independent; *auto-fixable*. Land before anything else.

**Batch B — Multi-repo migration fixes (`jobs.ts` / `jobsRepo.ts` / `config.ts`).** P0-5 multi-repo job loading, P1-10 single-source slug, plus config-parse dedupe (P2 `jobsRepo` double-parse). One owner, one module family. *Auto-fixable* but verify against `migrateTriggers.ts` contract.

**Batch C — Daemon decomposition + concurrency (`start.ts` / `hookQueue.ts` / `runner.ts` sessions).** P0-6 thread-keyed job state, P0-7 `runOutcome` skip detection, P0-14 coalesced-retry caps, P0-10 session-cache serialization, P1-5 `start()` extraction. *Risky* — concurrency semantics; do the `threadId`-keying and serialized-writer work together with tests. Human review on retry-policy ownership.

**Batch D — Hooks consolidation (`match.ts` / 4 receivers / `shared/hookPayload.ts`).** P0-8 glob-exclusion semantics, P1-6 `evalPrRule`-for-Sentry/Datadog + `handleSignedWebhook` envelope, plus P2 `readPath`/`pickPullRequest`/Linear-id dedup. Self-contained area, high leverage; *auto-fixable* but the glob-default flip needs a human decision on intended semantics.

**Batch E — Chat-integration abstraction (`telegram.ts` / `discord.ts` / `slack.ts` / `shared/`).** P1-1 `ChatPlatform` interface + `handleInboundMessage` orchestrator, P1-7 directive unification + capability/safety parity, P0-15 async/gated Discord classifier. The single largest structural win (~2,500 lines retired) and closes the Slack/Discord safety drift — but it's the *most invasive*; needs a human-blessed design pass and careful migration.

**Batch F — Web layer (`server.ts` routes + UI bundle/file decomposition).** P1-3 route-table extraction (subsumes P1-8 body-parse/try-catch/SSE/import dup), P1-11 `SettingsSection`/`RunsSection` split + `useServerSeed`, P0-12 `HookConfigEditor.emit`, P1-12 shared provider/trigger parser. **Includes the one explicit human decision: deleting `web/app`/`web/os9`/`web/osish` + build/server wiring (P1-2)** — confirm no bundle is wanted before removing ~6.2k LOC. Route extraction and bundle deletion can land in parallel (different files).
